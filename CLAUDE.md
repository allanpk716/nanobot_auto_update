# 项目规则和约定

本文档定义了项目的开发规范和约定，所有开发工作都应遵循这些规则。

## 语言和沟通

- **使用中文回答问题** - 所有交流使用中文进行
- **开发脚本禁止中文** - 开发、编辑 BAT 脚本时不包含任何中文字符

## 开发环境

- **操作系统**: Windows
- **临时文件**: 临时测试代码、数据统一放在项目根目录的 `tmp` 文件夹中

## 项目组织结构

- **计划文件**: 项目的计划文件存放在 `docs\plans` 目录
- **Bug 记录**: 项目自测发现的 bug 放在 `docs\bugs` 目录

## 脚本开发规范

- **修复优先于新建** - 修复脚本时在原有脚本基础上修改，非必需不新建脚本

## 图像处理

- **图片尺寸限制** - 使用截图 MCP 或 Agent 能力时，提交到识别图片服务前确保图片尺寸小于 1000x1000

## 前后端开发

- **集成测试** - 前后端分离项目使用浏览器（dev-browser 技能）测试前后端通信和交互操作

## Golang 开发规范

- **日志库**: 使用 `github.com/WQGroup/logger`
- **日志格式**: `2024-01-01 12:00:00.123 - [INFO]: 应用程序启动`
- **使用说明**: 使用 Context7 查询该日志库的使用方法

---

> 注意: 如需修改或添加新规则，请在对应分类下进行，避免重复和冲突。
