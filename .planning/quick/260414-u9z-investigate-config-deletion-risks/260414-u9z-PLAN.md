# Quick Task Plan: 调查配置文件删除风险

## 调查范围

扫描所有可能导致 nanobot 实例的 config.json 或其所在目录被清空、删除、覆盖的代码路径。

## 调查维度

1. **直接删除操作** - os.Remove / os.RemoveAll 调用链路
2. **覆盖写入操作** - os.WriteFile / WriteConfig 调用链路
3. **实例管理流程** - 创建/复制/更新/删除实例
4. **热重载流程** - config.yaml 变更触发的回调
5. **API 端点** - 前端可触发的配置修改操作
6. **自更新流程** - 应用更新对配置文件的影响
7. **边缘场景** - 共享配置路径、目录级操作、空 body 写入

## 调查文件

- internal/nanobot/config_manager.go
- internal/api/instance_config_handler.go
- internal/api/nanobot_config_handler.go
- internal/config/hotreload.go
- internal/config/config.go
- cmd/nanobot-auto-updater/main.go
- internal/api/server.go
