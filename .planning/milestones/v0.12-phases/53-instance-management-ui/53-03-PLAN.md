|-----------|----------|-----------|-------------|-----------------|
| T-53-10 | Tampering | Arbitrary JSON injection via textarea | mitigate | Server-side JSON parsing validates structure; file write limited to nanobot config path per instance; no code execution from config |
| T-53-11 | Information Disclosure | API keys visible in structured form | mitigate | API key input uses type="password" by default with show/hide toggle; still visible on localhost admin tool but not displayed in plaintext by default |
| T-53-12 | Denial of Service | Invalid nanobot config breaks instance | accept | User has explicit control; toast hints that restart is needed to apply; nanobot handles invalid config gracefully on startup |
| T-53-13 | Elevation of Privilege | Path traversal via config content | mitigate | Server-side nanobot config path is computed from instance config, not user input; write location is fixed per instance |
</threat_model>
