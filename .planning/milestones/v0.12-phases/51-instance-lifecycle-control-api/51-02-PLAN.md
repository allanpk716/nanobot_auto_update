```go
// internal/instance/lifecycle_test_helper.go
// Test-only helpers for InstanceLifecycle. Used by handler tests to inject running state
// without requiring a real process. This file is safe for production builds -- the helpers
// are only useful in test contexts and do not affect runtime behavior.

package instance

// SetPIDForTest sets the internal PID field for testing purposes.
// This allows tests to simulate a running instance without starting a real process.
// The PID value should be set to a non-zero value to make IsRunning() potentially return true.
// NOTE: IsRunning() also checks if the process exists via gopsutil, so the test must
// either use a PID that corresponds to an existing process (e.g., the test process itself)
// or use a PID that FindProcessByPID will find. The simplest approach: use os.Getpid()
// in the test to get the current test process PID, which always exists.
func (il *InstanceLifecycle) SetPIDForTest(pid int32) {
	il.pid = pid
}
```

Key points:
- This is a production-code file (no `_test.go` suffix) because `api` package tests need to call it from outside the `instance` package.
- It only sets `il.pid`. The `IsRunning()` method will still call `lifecycle.FindProcessByPID`, so the test must use a real PID (e.g., `os.Getpid()` for the test process itself).
- The method name `SetPIDForTest` makes the intent obvious -- it is never called in production code.
- This is the minimal change approach (review suggestion: "code-level state injection") -- avoids creating an interface abstraction for a single test need.
