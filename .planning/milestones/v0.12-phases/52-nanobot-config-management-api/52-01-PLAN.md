**HandleGet** handles `GET /api/v1/instance-configs/{name}/nanobot-config` (NC-02):
1. Extract name from `r.PathValue("name")`
2. Get current config via `h.getConfig()`, check for nil
3. Find instance via `findInstanceByName(cfg, name)`, return 404 if not found
4. Parse config path via `nanobot.ParseConfigPath(ic.StartCommand, ic.Name)`
5. Read config via `h.manager.ReadConfig(configPath)`
6. **LAZY-CREATION FALLBACK** (addresses review concern: inconsistent state from failed callbacks): If file not found (os.IsNotExist), auto-create a default nanobot config for this instance:
   ```go
   if os.IsNotExist(err) {
       h.logger.Warn("Nanobot config missing for known instance, auto-creating default config",
           "instance", ic.Name, "path", configPath)
       if createErr := h.manager.CreateDefaultConfig(ic.Name, ic.Port, ic.StartCommand); createErr != nil {
           h.logger.Error("Failed to auto-create nanobot config", "instance", ic.Name, "error", createErr)
           writeJSONError(w, http.StatusInternalServerError, "internal_error", "Failed to create nanobot config")
           return
       }
       // Re-read the freshly created config
       configData, err = h.manager.ReadConfig(configPath)
       if err != nil {
           writeJSONError(w, http.StatusInternalServerError, "internal_error", "Failed to read nanobot config")
           return
       }
   }
   ```
7. Return 200 with the config JSON as response body. Wrap in object: `{"config": <config map>, "instance": name}`
8. On other errors, return 500

**HandlePut** handles `PUT /api/v1/instance-configs/{name}/nanobot-config` (NC-03, D-13):
1. Extract name from `r.PathValue("name")`
2. Get current config, check for nil
3. Find instance, return 404 if not found
4. Decode request body into `map[string]interface{}` via `json.NewDecoder(r.Body).Decode(&reqBody)`
5. Validate decoded body is valid JSON (Decode returns error if not) -- if error, return 400 "Invalid JSON body"
6. Parse config path via `nanobot.ParseConfigPath(ic.StartCommand, ic.Name)`
7. Call `h.manager.WriteConfig(configPath, reqBody)` (mutex-protected)
8. On success, return 200 with `{"message": "Nanobot config updated for instance {name}", "instance": name}`
9. D-14: Include informational hint: `"hint": "Restart the instance via POST /api/v1/instances/{name}/stop + POST /api/v1/instances/{name}/start to apply changes"`
10. On write error, return 500
