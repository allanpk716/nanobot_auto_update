Create file `.goreleaser.yaml` at project root with the following exact content:

```yaml
# GoReleaser configuration for nanobot-auto-updater
# Builds Windows amd64 GUI binary and publishes to GitHub Releases

before:
  hooks:
    - go mod tidy

builds:
  - main: ./cmd/nanobot-auto-updater
    binary: nanobot-auto-updater
    goos:
      - windows
    goarch:
      - amd64
    # IMPORTANT: Must fully override GoReleaser default ldflags.
    # Default uses lowercase main.version, project uses uppercase main.Version.
    # -H=windowsgui matches Makefile LDFLAGS_RELEASE (per D-02: GUI-only build).
    ldflags:
      - -H=windowsgui -X main.Version={{.Version}}
    env:
      - CGO_ENABLED=0

archives:
  - format: zip
    # Output: nanobot-auto-updater_1.0.0_windows_amd64.zip
    # Per D-01: ZIP format for Phase 38 self-update to download and extract exe.
    name_template: >-
      {{ .ProjectName }}_
      {{- .Version }}_
      {{- .Os }}_
      {{- .Arch }}

checksum:
  # Output: nanobot-auto-updater_1.0.0_checksums.txt
  # SHA256 checksums needed by Phase 38 UPDATE-03 integrity verification.
  name_template: "{{ .ProjectName }}_{{ .Version }}_checksums.txt"
  algorithm: sha256
```

Key configuration decisions:
- Single platform: windows/amd64 only (per D-02: only GUI version)
- ZIP archive format (per D-01: Phase 38 will download ZIP and extract exe)
- ldflags fully overridden to use uppercase `main.Version` matching Makefile LDFLAGS_RELEASE pattern
- CGO_ENABLED=0 for clean cross-compilation on ubuntu runner
- SHA256 checksums for Phase 38 integrity verification

Create directory `.github/workflows/` and file `.github/workflows/release.yml` with the following exact content:

```yaml
name: goreleaser

on:
  push:
    tags:
      - "v*"

permissions:
  contents: write

jobs:
  goreleaser:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version: stable

      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v7
        with:
          distribution: goreleaser
          version: "~> v2"
          args: release --clean
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

Key configuration decisions:
- Trigger: `on: push: tags: - "v*"` (per CICD-01)
- Single job with GoReleaser managing everything (per D-03: GoReleaser 管一切)
- fetch-depth: 0 required for GoReleaser changelog generation and version detection
- permissions: contents: write required for GITHUB_TOKEN to create Releases
- go-version: stable uses latest stable Go, appropriate for go 1.24.11 project
- version: "~> v2" tracks latest v2.x patch, avoids unexpected major version bumps
- No separate test job (per D-03: keep workflow simple)
