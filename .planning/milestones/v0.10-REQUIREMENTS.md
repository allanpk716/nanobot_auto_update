# v0.10 Requirements: 管理界面自更新功能

## Milestone Goal

在现有嵌入式 Web UI (home.html) 顶部新增自更新管理区域，允许用户通过浏览器检查最新版本、查看版本详情并触发自动更新，实时显示更新进度（含下载百分比）。

---

## Requirements

### UI-01: 自更新管理区域布局
- home.html 页面顶部（实例列表上方）新增"自更新管理"区块
- 区块包含：当前版本号、"检测更新"按钮、"立即更新"按钮（初始禁用）
- 区块使用与现有 UI 一致的 CSS 风格（style.css 变量、颜色、按钮样式）
- 默认收起/折叠状态，点击展开显示详情（可选，优先始终展开）

### UI-02: 当前版本显示
- 区块内始终显示当前运行版本号（从 API 获取）
- 版本号以标签样式展示（如 `v0.9.0`）

### UI-03: 检测更新
- 点击"检测更新"调用 `GET /api/v1/self-update/check`
- 显示最新版本号、发布日期、更新说明（release notes）
- 已是最新版本时显示"已是最新版本"提示
- 有新版本时显示详情并启用"立即更新"按钮

### UI-04: 触发更新与进度显示
- 点击"立即更新"调用 `POST /api/v1/self-update`
- 更新过程显示进度状态，阶段包括：
  - **检查中** (checking)
  - **下载中** (downloading) — 显示进度百分比
  - **安装中** (installing)
  - **完成** (complete)
  - **失败** (failed) — 显示错误信息
- 更新进行中禁用所有操作按钮，防止重复触发
- 更新完成后显示成功提示

### UI-05: 下载进度百分比
- 下载阶段实时显示进度百分比（如 "下载中 45%"）
- 前端通过轮询方式获取进度（约 500ms 间隔）
- 进度以进度条 + 百分比文字形式展示

### API-01: 更新进度状态追踪
- selfupdate 包新增进度状态结构：
  - `stage`: idle / checking / downloading / installing / complete / failed
  - `download_percent`: 0-100 整数
  - `error`: 错误信息（仅 failed 状态）
- 下载时使用 `io.TeeReader` + `Content-Length` 追踪已下载/总字节数
- 进度状态存储为 `atomic.Value`，并发安全
- `GET /api/v1/self-update/check` 响应新增 `progress` 字段

### API-02: Web UI Token API
- 新增 `GET /api/v1/web-config` 端点，返回 Web UI 所需配置
- 响应包含 `auth_token` 字段（从配置文件读取）
- 仅允许 localhost (127.0.0.1 / ::1) 访问，非 localhost 返回 403
- 前端 JS 在页面加载时自动获取 token，存储后用于自更新 API 调用

---

## Out of Scope

- 版本更新历史记录页面
- 自动定时检测更新（仅手动触发）
- 多语言支持（保持中文）
- 更新回滚 UI（已有后端 .old 恢复机制）

---

## Success Criteria

1. home.html 顶部自更新区域正确显示当前版本
2. "检测更新"按钮能获取并展示最新版本详情（版本号、日期、说明）
3. "立即更新"按钮能触发自更新流程
4. 更新过程中实时显示阶段和下载百分比
5. 认证 token 从配置自动获取，无需手动输入
6. 所有新增后端代码有单元测试覆盖
