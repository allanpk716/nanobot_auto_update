## E2E Flow Verification

### Flow 1: User Opens Log Viewer
**Steps:**
1. Browser requests /logs/{instance}
2. Web handler serves embedded HTML
3. JavaScript connects to SSE endpoint
4. SSE handler subscribes to LogBuffer
5. History logs sent first
6. Real-time logs streamed as they arrive

**Status:** ✓ COMPLETE
**Requirements:** UI-01, UI-05, SSE-01, SSE-02, SSE-05
**Evidence:** All code paths verified in integration checker

### Flow 2: Instance Produces Logs
**Steps:**
1. Process stdout/stderr captured by os.Pipe
2. Concurrent goroutines call captureLogs
3. captureLogs writes to LogBuffer with source tag
4. LogBuffer.Write sends to subscribers
5. SSE handler forwards with event type
6. JavaScript appends with correct CSS class

**Status:** ✓ COMPLETE
**Requirements:** CAPT-01/02/03, BUFF-04, SSE-06, UI-04
**Evidence:** starter.go:156-189, capture.go:47-52, sse.go:110-113, app.js:143

### Flow 3: User Switches Instances
**Steps:**
1. User selects instance in dropdown
2. Old EventSource closed
3. Log container cleared
4. URL updated
5. New SSE connection established
6. New instance logs appear

**Status:** ✓ COMPLETE
**Requirements:** UI-07, UI-02
**Evidence:** app.js:52-70, handler.go:73

### Flow 4: Process Lifecycle
**Steps:**
1. Process exits
2. cmd.Wait returns
3. Pipe writers closed
4. cancelCapture() stops goroutines
5. LogBuffer preserved on stop
6. LogBuffer cleared on restart
7. New process starts with fresh buffer

**Status:** ✓ COMPLETE
**Requirements:** CAPT-05, INST-04, INST-05
**Evidence:** starter.go:207-220, lifecycle.go:43-79, 90
