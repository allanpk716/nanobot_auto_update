## E2E Flows

### Flow 1: Application Startup and Auto-Start

**Steps:**
1. Config loaded
2. InstanceManager created
3. API server started
4. HealthMonitor started
5. NetworkMonitor started
6. NotificationManager started
7. Auto-start triggered
8. Instances check ShouldAutoStart()
9. Instances started sequentially
10. HealthMonitor checks started instances

**Status:** ✓ COMPLETE - All components start in correct order, auto-start works correctly

### Flow 2: Network State Change Notification

**Steps:**
1. NetworkMonitor checks Google connectivity
2. State updated with RWMutex protection
3. NotificationManager polls state
4. State change detected
5. Cooldown timer started (1 minute)
6. Timer fires
7. State confirmed
8. Notification sent via Pushover

**Status:** ✓ COMPLETE - Network state changes trigger notifications after cooldown

### Flow 3: HTTP API Trigger Update

**Steps:**
1. Client sends POST /api/v1/trigger-update with Bearer token
2. AuthMiddleware validates token
3. TriggerHandler.Handle called
4. TriggerUpdate called with concurrent check
5. UpdateAll executes stop->update->start flow
6. Result returned as JSON

**Status:** ✓ COMPLETE - HTTP API trigger correctly executes update flow with concurrent control

### Flow 4: Application Graceful Shutdown

**Steps:**
1. Shutdown signal received (SIGINT/SIGTERM)
2. NotificationManager stopped
3. NetworkMonitor stopped
4. HealthMonitor stopped
5. API server shutdown
6. All goroutines exit via context cancellation

**Status:** ✓ COMPLETE - Graceful shutdown in correct reverse order

### Flow 5: Health Monitoring and State Detection

**Steps:**
1. HealthMonitor ticker fires
2. Check all instances
3. IsNanobotRunning called
4. State change detected
5. ERROR or INFO logged based on transition

**Status:** ✓ COMPLETE - Health monitoring detects and logs state changes correctly
