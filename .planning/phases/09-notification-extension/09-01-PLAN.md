   - 第三部分: 启动失败详情 (如果有):
     * 标题 "启动失败的实例:\n"
     * 遍历 result.StartFailed,格式化为 "  ✗ {name} (端口 {port})\n    原因: {err}\n"
   - 第四部分: 成功启动列表 (如果有):
     * 标题 "成功启动的实例 ({count}):\n"
     * 遍历 result.Started,格式化为 "  ✓ {name}\n"
   - 返回 msg.String()

3. 创建 internal/notifier/notifier_ext_test.go:
   - TestNotifyUpdateResult_NoErrors: 验证所有成功时不发送通知
   - TestNotifyUpdateResult_WithStopFailures: 验证停止失败时发送通知
   - TestNotifyUpdateResult_WithStartFailures: 验证启动失败时发送通知
   - TestNotifyUpdateResult_WithMixedResults: 验证混合结果时消息完整性
   - TestFormatUpdateResultMessage_Formatting: 验证消息格式化细节

技术要点:
- 使用 fmt.Sprintf("%v", err.Err) 格式化底层错误,避免技术错误码
- 消息中使用 ✗ (U+2717) 和 ✓ (U+2713) Unicode 符号提升可读性
- 每个部分之间使用 "\n\n" 分隔,列表项之间使用 "\n"
- DEBUG 日志记录: "All instances succeeded, skipping failure notification", 包含 stopped/started 计数
- 参考 RESEARCH.md 的 Architecture Patterns 部分实现代码

避免:
- 不要为每个失败实例发送单独通知 (通知风暴反模式)
- 不要在所有实例成功时发送通知 (不必要的打扰反模式)
- 不要只显示失败实例不显示成功实例 (不完整状态反模式)
- 不要使用 yaml 标签访问 UpdateResult 字段 (它是结构体不是配置)
