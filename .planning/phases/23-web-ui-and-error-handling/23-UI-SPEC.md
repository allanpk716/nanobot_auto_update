# Phase 23 — UI Design Contract

> Visual and interaction contract for the embedded Web UI log viewer. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native HTML/CSS/JS) |
| Preset | not applicable |
| Component library | none |
| Icon library | none |
| Font | system-ui (browser default) |

**Rationale:** Phase 23 是一个简单的嵌入式日志查看器（约 300 行 JS），使用原生 Web 技术以实现单文件部署（Go embed.FS）。无需前端框架或设计系统。

|------|------|--------|-------------|
| Body | 14px | 400 (normal) | 1.5 |
| Label | 14px | 600 (semibold) | 1.2 |

**Rationale:**
- 单一字号（14px）适合日志查看器密集信息展示

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff | 主背景色，日志容器背景 |
| Secondary (30%) | #f5f5f5 | 头部、状态栏背景 |
| Accent (10%) | #2563eb (blue-600) | **仅用于：连接状态指示器（已连接）、按钮 hover 状态** |
| Destructive | #dc2626 (red-600) | stderr 日志文本、断开连接状态指示器 |

**Log source differentiation (UI-04):**
- **stdout:** `color: #000000` (黑色，正常文本)
- **stderr:** `color: #dc2626; font-weight: 600` (红色加粗，高对比度)

Accent reserved for: 连接状态指示器（connected 状态）、按钮交互状态

**Rationale:** 遵循 60/30/10 配色法则，使用高对比度红色区分 stderr（RESEARCH.md Pitfall 4），避免色盲用户无法识别错误日志。

## Interaction Patterns

### Pattern 1: Auto-Scroll with Smart Toggle
**What:** 默认自动滚动到最新日志，用户手动滚动时暂停
**When to use:** 实时日志流场景（UI-02, UI-03）
**Behavior:**
- Initial state: `autoScroll = true`
- User scrolls up: `autoScroll = false` (if >50px from bottom)
- User clicks "恢复滚动": `autoScroll = true`, scroll to bottom
- New log arrives: only scroll if `autoScroll === true`

### Pattern 2: SSE Connection Status
**What:** 实时显示连接状态，断线时自动重连
**When to use:** SSE 长连接场景（UI-05）
**Behavior:**
- `onopen`: 显示绿色 "已连接"
- `onerror`: 显示红色 "已断开"，EventSource 自动重连（3s delay）
- Custom logic: 10 次重连失败后显示 "请刷新页面"

### Pattern 3: Instance Switching
**What:** 下拉菜单切换实例，自动重新连接 SSE
**When to use:** 多实例场景（UI-07）
**Behavior:**
- Page load: fetch `/api/v1/instances` → populate dropdown
- User selects instance: close existing EventSource → clear logs → connect to new SSE stream
- URL pattern: `/logs/{instance}` (from URL param, not hardcoded)

## Error Handling Contract

| Error Scenario | UI Behavior | Backend Behavior |
|----------------|-------------|------------------|
| Instance not found (UI-01) | Display "实例不存在" error message | HTTP 404 Not Found |
| SSE connection failed (ERR-02) | Show "已断开" status, auto-reconnect | Log warning, continue serving |
| Pipe read error (ERR-01) | N/A (backend handles) | Log error, continue capture |
| LogBuffer write error (ERR-03) | N/A (backend handles) | Log error, drop log line |
