         * 使用 context.WithTimeout() 创建带超时的 ctx

         * 双层错误检查:
           - 如果 err != nil (UV 更新失败): 调用 notif.NotifyFailure("Multi-Instance Update", err),输出 JSON 并 os.Exit(1)
           - 如果 result.HasErrors() (实例失败): 调用 notif.NotifyUpdateResult(result),输出成功 JSON 但包含失败数量
           - 否则: 输出成功 JSON
         * 使用 os.Exit(0) 退出
       - 如果是 legacy 模式: 保持现有逻辑不变

    4. 定时任务模式分支 (L238-L261):
       - 检测 useMultiInstance 标志
       - 如果是多实例模式:
         * 创建 InstanceManager: `manager := instance.NewInstanceManager(cfg, logger)`
         * 在 sched.AddJob() 中使用 context.Background()
         * 调用 `result, err := manager.UpdateAll(ctx)`
         * 双层错误检查:
           - 如果 err != nil: 调用 notif.NotifyFailure("Scheduled Multi-Instance Update", err),return
           - 如果 result.HasErrors(): 调用 notif.NotifyUpdateResult(result),记录 ERROR 日志
           - 否则: 记录 INFO 日志显示成功停止和启动的实例数量
       - 如果是 legacy 模式: 保持现有逻辑不变

    5. 保持向后兼容:
       - legacy 模式使用 lifecycle.Manager,不引入 instance 包
       - 多实例模式使用 instance.InstanceManager,不使用 lifecycle.Manager

    关键注意事项:
    - 定时任务使用 context.Background(),不要使用 WithTimeout()
    - --update-now 使用 context.WithTimeout(context.Background(), *timeout)
    - 双层错误检查: UV 更新失败 (err) + 实例失败 (result.HasErrors())
    - 使用 result.Stopped/Started 的长度记录成功日志
    - LIFECYCLE-03 (优雅降级) 和 ERROR-02 (错误聚合) 已由 Phase 8 InstanceManager 实现,本任务集成使用
