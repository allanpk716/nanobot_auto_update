# Plan 33-01: End-to-End Integration Tests

## Goal

Verify the complete update log flow: trigger → file persistence → query retrieval, and update ID consistency.

## Success Criteria Covered

- SC-1: POST /api/v1/trigger-update triggers update and records log to file
- SC-2: GET /api/v1/update-logs returns the recently recorded update
- SC-3: Log recording failure does not affect update operation (file write failure test)
- SC-5: Update ID is consistent between trigger response and query results

## Tasks

### Task 1: Create E2E integration test for trigger → file → query flow

Create `internal/api/integration_test.go`:

1. **TestE2E_TriggerUpdate_RecordsTo_QueryReturns**: Full chain test
   - Create UpdateLogger with temp JSONL file path
   - Create TriggerHandler with mock TriggerUpdater (successful result with Stopped/Started)
   - Create QueryHandler sharing the same UpdateLogger
   - Call TriggerHandler.Handle() → verify 200 OK with update_id
   - Call QueryHandler.Handle() → verify 200 OK with data containing the same update_id
   - Verify JSONL file was created and contains 1 line with the update_id
   - Cleanup temp file

2. **TestE2E_UpdateID_Consistency**: Verify update_id match
   - Same setup as above but with 3 sequential trigger calls
   - Collect all 3 update_ids from trigger responses
   - Query update-logs and verify all 3 IDs appear in results
   - Verify query returns them in newest-first order

3. **TestE2E_NonBlocking_FileWriteFailure**: Verify non-blocking on invalid path
   - Create UpdateLogger with invalid file path (e.g., non-existent deep directory)
   - Create TriggerHandler with mock (successful result)
   - Call Handle() → verify 200 OK (update succeeds despite file write failure)
   - Verify in-memory log was still recorded (GetAll returns 1 entry)

4. **TestE2E_LoadFromFile_StartupRecovery**: Verify startup history recovery
   - Write 3 JSONL lines manually to a temp file
   - Create UpdateLogger with that file path
   - Call LoadFromFile()
   - Call QueryHandler.Handle() → verify all 3 records returned
   - Trigger a new update via TriggerHandler
   - Query again → verify 4 records total (3 loaded + 1 new)

### Task 2: Verify all tests pass

Run `go test ./internal/api/... ./internal/updatelog/...` and ensure all tests pass.
