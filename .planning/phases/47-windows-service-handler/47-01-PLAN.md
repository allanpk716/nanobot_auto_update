type AppComponents struct {

    CleanupCron         *cron.Cron

Define AppShutdown function (D-05, D-07) -- shutdown order MUST match main.go lines 288-317 exactly:
```go
// AppShutdown performs ordered shutdown of all non-nil components (D-05, D-07).
// Components are shut down in the same order as the original main.go:
// notificationManager -> networkMonitor -> healthMonitor -> cleanupCron -> updateLogger -> apiServer.
// Internal dependencies (Notifier, InstanceManager, SelfUpdater) do not have Stop/Close methods.
func AppShutdown(ctx context.Context, c *AppComponents, logger *slog.Logger) {
    if c == nil {
        return
    }
    if c.NotificationManager != nil {
        c.NotificationManager.Stop()
    }
    if c.NetworkMonitor != nil {
        c.NetworkMonitor.Stop()
    }
    if c.HealthMonitor != nil {
        c.HealthMonitor.Stop()
    }
    if c.CleanupCron != nil {
        c.CleanupCron.Stop()
        logger.Info("Update log cleanup scheduler stopped")
    }
    if c.UpdateLogger != nil {
        if err := c.UpdateLogger.Close(); err != nil {
            logger.Error("Failed to close update logger", "error", err)
        }
    }
    if c.APIServer != nil {
        if err := c.APIServer.Shutdown(ctx); err != nil {
            logger.Error("API server shutdown error", "error", err)
        }
    }
    logger.Info("Shutdown completed")
}
```

Define AppStartup function (D-10) with explicit ROLLBACK on error (addresses review concern: AppStartup error/rollback contract undefined):
```go
// AppStartup initializes all application components and returns a container (D-10).
// On error, any partially-initialized components are cleaned up via AppShutdown (rollback).
// The caller should NOT call AppShutdown again on error -- rollback is already done.
// On success, the caller is responsible for calling AppShutdown when done.
func AppStartup(cfg *config.Config, logger *slog.Logger, version string) (*AppComponents, error) {
    c := &AppComponents{
        AutoStartDone: make(chan struct{}),
    }

    // Helper: on error, rollback partial components and return
    // Each step that can fail should use this pattern:
    //   if err != nil {
    //       AppShutdown(ctx, c, logger)
    //       return nil, err
    //   }

    rollback := func(err error) (*AppComponents, error) {
        cleanupCtx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
        AppShutdown(cleanupCtx, c, logger)
        cancel()
        return nil, err
    }

    // Step 1: UpdateLogger (always created, never fails constructor)
    c.UpdateLogger = updatelog.NewUpdateLogger(logger, "./logs/updates.jsonl")

    // Step 2: CleanupOldLogs + LoadFromFile (non-fatal, log errors)
    if err := c.UpdateLogger.CleanupOldLogs(); err != nil {
        logger.Error("Failed to cleanup old update logs", "error", err)
    }
    if err := c.UpdateLogger.LoadFromFile(); err != nil {
        logger.Error("Failed to load update logs from file", "error", err)
    }

    // Step 3: CleanupCron (cron.New() never fails, AddFunc/Start never fail)
    c.CleanupCron = cron.New()
    c.CleanupCron.AddFunc("0 3 * * *", func() {
        if err := c.UpdateLogger.CleanupOldLogs(); err != nil {
            logger.Error("Scheduled update log cleanup failed", "error", err)
        }
    })
    c.CleanupCron.Start()
    logger.Info("Update log cleanup scheduler started", "schedule", "0 3 * * *")

    // Step 4: Notifier (always created, handles disabled state internally)
    c.Notifier = notifier.NewWithConfig(
        notifier.Config{
            ApiToken: cfg.Pushover.ApiToken,
            UserKey:  cfg.Pushover.UserKey,
        },
        logger,
    )

    // Step 5: InstanceManager (needs Notifier)
    c.InstanceManager = instance.NewInstanceManager(cfg, logger, c.Notifier)

    // Step 6: SelfUpdater
    c.SelfUpdater = selfupdate.NewUpdater(
        selfupdate.SelfUpdateConfig{
            GithubOwner: cfg.SelfUpdate.GithubOwner,
            GithubRepo:  cfg.SelfUpdate.GithubRepo,
        },
        logger,
    )

    // Step 7: APIServer (conditional, can fail)
    if cfg.API.Port != 0 {
        var err error
        c.APIServer, err = api.NewServer(&cfg.API, c.InstanceManager, cfg, version, logger, c.UpdateLogger, c.Notifier, c.SelfUpdater)
        if err != nil {
            logger.Error("Failed to create API server", "error", err)
            return rollback(err)
        }
        go func() {
            logger.Info("API server starting", "port", cfg.API.Port)
            if err := c.APIServer.Start(); err != nil {
                logger.Error("API server error", "error", err)
            }
        }()
    }

    // Step 8: HealthMonitor (conditional, no failure path)
    if len(cfg.Instances) > 0 {
        c.HealthMonitor = health.NewHealthMonitor(
            cfg.Instances,
            cfg.HealthCheck.Interval,
            logger,
        )
        go c.HealthMonitor.Start()
        logger.Info("Health monitor started", "interval", cfg.HealthCheck.Interval)
    }

    // Step 9: NetworkMonitor (always created, no failure path)
    c.NetworkMonitor = network.NewNetworkMonitor(
        "https://www.google.com",
        cfg.Monitor.Interval,
        cfg.Monitor.Timeout,
        logger,
    )
    go c.NetworkMonitor.Start()
    logger.Info("Network monitor started", "interval", cfg.Monitor.Interval)

    // Step 10: NotificationManager (always created, no failure path)
    c.NotificationManager = notification.NewNotificationManager(
        c.NetworkMonitor,
        c.Notifier,
        logger,
    )
    go c.NotificationManager.Start(cfg.Monitor.Interval)
    logger.Info("Notification manager started", "check_interval", cfg.Monitor.Interval)

    // Step 11: Auto-start goroutine (addresses review concern: auto-start goroutine ownership)
    // AppStartup owns launching this goroutine. The AutoStartDone channel is
    // closed when the goroutine finishes, allowing callers to wait if needed.
    go func() {
        defer close(c.AutoStartDone)
        defer func() {
            if r := recover(); r != nil {
                logger.Error("Auto-start goroutine panic",
                    "panic", r,
                    "stack", string(debug.Stack()))
            }
        }()

        autoStartTimeout := 5 * time.Minute
        autoStartCtx, cancel := context.WithTimeout(context.Background(), autoStartTimeout)
        defer cancel()

        logger.Info("Starting auto-start for all instances",
            "instance_count", len(cfg.Instances),
            "timeout", autoStartTimeout)

        result := c.InstanceManager.StartAllInstances(autoStartCtx)
        if err := c.Notifier.NotifyStartupResult(result); err != nil {
            logger.Error("Failed to send startup notification", "error", err)
        }
    }()

    return c, nil
}
```

Key design decisions:
- AppComponents includes Notifier, InstanceManager, SelfUpdater as fields (addresses review HIGH: missing key dependencies). These are internal dependencies needed by apiServer constructor and auto-start goroutine. They do NOT have Stop/Close methods so they are NOT in the shutdown sequence.
- Rollback: When AppStartup fails (currently only at api.NewServer), the `rollback` helper calls AppShutdown on partial components, then returns (nil, error). The caller does NOT need to call AppShutdown again on error. This prevents resource leaks.
- AutoStartDone channel: The auto-start goroutine closes this channel when done. ServiceHandler can optionally wait on it during shutdown if needed. For testing, this channel indicates when startup is complete.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AppStartup -> Component constructors | AppStartup calls constructors for all components. Constructor failures trigger rollback. |
| AppShutdown -> Component Stop/Close | Shutdown calls Stop/Close methods. All calls are nil-safe. 30s context timeout prevents hangs. |
| Config -> AppStartup | AppStartup reads cfg for component parameters. Config already validated by config.Load(). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-47-01 | Denial of Service | AppShutdown | mitigate | 30-second context timeout ensures shutdown completes even if a component hangs. APIServer.Shutdown(ctx) respects the context. Other Stop() calls are near-instant. |
| T-47-02 | Denial of Service | AppStartup rollback | mitigate | Rollback uses 30-second context timeout. If partial cleanup hangs, the process will be killed by OS (SCM or user). |
| T-47-03 | Tampering | Config -> AppStartup | accept | Config already validated by config.Load() with strict validation rules. AppStartup does not modify config. |
</threat_model>
