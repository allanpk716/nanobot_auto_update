**Method: `RegisterService() error`** (per D-01, D-02, D-03, D-04, D-07)
Steps in order:
1. **Defensive check (addresses review concern: empty ServiceName):** If `cfg.Service.ServiceName == ""`, return `fmt.Errorf("registerService: service_name is empty, cannot register service")`.
2. Call `mgr.Connect()` to connect to SCM. If error, wrap with `fmt.Errorf("registerService: failed to connect to SCM: %w", err)` and return.
3. Defer `m.Disconnect()`.
4. Try `m.OpenService(cfg.Service.ServiceName)` to check if service already exists.
5. If `OpenService` succeeds (service exists per D-04): close the service handle, log "service already registered, skipping" with service_name, and return nil (idempotent).
6. If `OpenService` fails (service does not exist): proceed to create.
7. Get executable path: `os.Executable()` -- this is the binary to register. Log the full exe path for auditability (addresses review concern): `logger.Info("Registering service", "exe_path", exePath, "service_name", cfg.Service.ServiceName)`.
8. Call `m.CreateService()` with these exact parameters per the decisions:
   - `name`: `cfg.Service.ServiceName`
   - `exepath`: the executable path from step 7
   - `c`: `mgr.Config{
       StartType:        mgr.StartAutomatic,        // D-02: auto start
       ErrorControl:     mgr.ErrorNormal,
       ServiceStartName: "LocalSystem",              // D-01: LocalSystem account
       DisplayName:      cfg.Service.DisplayName,
       Description:      "自动保持 nanobot 处于最新版本", // D-03: hardcoded Chinese
     }`
9. If CreateService fails, return wrapped error: `fmt.Errorf("registerService: failed to create service %q: %w", cfg.Service.ServiceName, err)`.
10. Defer `svcHandle.Close()`.
11. Configure recovery policy (D-07): call `svcHandle.SetRecoveryActions()` with 3 RecoveryAction entries (first failure, second failure, subsequent failures -- all ServiceRestart with 60 second delay), and resetPeriod = 86400 (24 hours in seconds):
    ```go
    recoveryActions := []mgr.RecoveryAction{
        {Type: mgr.ServiceRestart, Delay: 60 * time.Second},
        {Type: mgr.ServiceRestart, Delay: 60 * time.Second},
        {Type: mgr.ServiceRestart, Delay: 60 * time.Second},
    }
    err = svcHandle.SetRecoveryActions(recoveryActions, 86400)
    ```
12. If SetRecoveryActions fails, return wrapped error: `fmt.Errorf("registerService: failed to set recovery actions for service %q: %w", cfg.Service.ServiceName, err)`.
13. Also call `svcHandle.SetRecoveryActionsOnNonCrashFailures(true)` so recovery triggers on non-zero exit codes too. If this fails, log a warning but do NOT return error (non-critical -- addresses review concern about error quality):
    ```go
    if err := svcHandle.SetRecoveryActionsOnNonCrashFailures(true); err != nil {
        m.logger.Warn("Failed to enable recovery on non-crash failures (non-critical)", "error", err)
    }
    ```
14. Log success: "Windows service registered successfully" with service_name and display_name.
15. Return nil.

**Method: `UnregisterService(ctx context.Context) error`** (per D-05, D-06; addresses review concern: context.Context for cancellation)
Steps in order:
1. Call `mgr.Connect()`. If error, wrap with `fmt.Errorf("unregisterService: failed to connect to SCM: %w", err)` and return.
2. Defer `m.Disconnect()`.
3. Call `m.OpenService(cfg.Service.ServiceName)`. If error (service not found), log "service not registered, nothing to uninstall" and return nil (idempotent).
4. If service found: try to stop it first (D-05). Call `svcHandle.Control(svc.Stop)` to send Stop control code. Log "stopping service before uninstall". If the Control call returns error, log warning but continue with delete -- the service may already be stopped.
5. After sending Stop, poll `svcHandle.Query()` in a loop (every 1 second, max 30 seconds) waiting for state `svc.Stopped`. **Use ctx for cancellation (addresses review concern):**
    ```go
    pollTicker := time.NewTicker(1 * time.Second)
    defer pollTicker.Stop()
    pollTimeout := time.After(30 * time.Second)
    for {
        select {
        case <-ctx.Done():
            m.logger.Warn("UnregisterService cancelled while waiting for service to stop", "error", ctx.Err())
            // Continue with delete despite cancellation
            goto deleteService
        case <-pollTimeout:
            m.logger.Warn("Timed out waiting for service to stop, proceeding with delete")
            goto deleteService
        case <-pollTicker.C:
            status, err := svcHandle.Query()
            if err != nil {
                m.logger.Warn("Failed to query service status, proceeding with delete", "error", err)
                goto deleteService
            }
            if status.State == svc.Stopped {
                m.logger.Info("Service stopped successfully")
                goto deleteService
            }
        }
    }
    ```
6. `deleteService:` label -- call `svcHandle.Delete()` to mark for deletion. If error, return wrapped error: `fmt.Errorf("unregisterService: failed to delete service %q: %w", cfg.Service.ServiceName, err)`. Log "service unregistered" with service_name.
7. Close the service handle.
8. Return nil. The caller (main.go) continues to console mode (D-06).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User config -> ServiceManager | Untrusted input from config.yaml (service_name, display_name) crosses into SCM API calls |
| ServiceManager -> Windows SCM | Process communicates with SCM (CreateService, DeleteService) requiring admin privileges |
| ServiceManager -> Filesystem | os.Executable() reads process binary path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48-01 | E (Elevation of Privilege) | RegisterService | mitigate | D-08: IsAdmin() checks for admin elevation before SCM operations. Non-admin gets clear error message and exits with code 1 |
| T-48-02 | S (Spoofing) | service_name config | mitigate | Phase 46 D-10: ServiceConfig.Validate() enforces alphanumeric-only service_name via regex, max 256 chars. Prevents injection into SCM API |
| T-48-03 | T (Tampering) | BinaryPathName | accept | os.Executable() returns current process path (kernel-provided). Attacker would need code execution already. Low risk for service registration |
| T-48-04 | D (Denial of Service) | UnregisterService | mitigate | context.Context parameter allows cancellation of 30s stop-wait loop. Without ctx, an unresponsive service could block indefinitely |
| T-48-05 | R (Repudiation) | Service operations | accept | Windows SCM logs all service create/delete to System event log. Built-in audit trail |
| T-48-06 | D (Denial of Service) | Empty ServiceName | mitigate | Defensive check at top of RegisterService rejects empty ServiceName before SCM API call. Prevents accidental SCM misuse |
</threat_model>
