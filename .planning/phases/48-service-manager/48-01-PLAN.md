**Method: `IsAdmin() bool`** (per D-08)
- Call `windows.OpenCurrentProcessToken()` to get the current process token
- Call `token.IsElevated()` to check admin elevation
- If `OpenCurrentProcessToken()` returns error, return false
- Close the token handle after check
- Implementation:
```go
func IsAdmin() bool {
    token, err := windows.OpenCurrentProcessToken()
    if err != nil {
        return false
    }
    defer token.Close()
    return token.IsElevated()
}
```

**Method: `RegisterService() error`** (per D-01, D-02, D-03, D-04, D-07)
Steps in order:
1. Call `mgr.Connect()` to connect to SCM. If error, wrap with `fmt.Errorf("failed to connect to SCM: %w", err)` and return.
2. Defer `m.Disconnect()`.
3. Try `m.OpenService(cfg.Service.ServiceName)` to check if service already exists.
4. If `OpenService` succeeds (service exists per D-04): close the service handle, log "service already registered, skipping" with service_name, and return nil (idempotent).
5. If `OpenService` fails (service does not exist): proceed to create.
6. Get executable path: `os.Executable()` -- this is the binary to register.
7. Call `m.CreateService()` with these exact parameters per the decisions:
   - `name`: `cfg.Service.ServiceName`
   - `exepath`: the executable path from step 6
   - `c`: `mgr.Config{
       StartType:        mgr.StartAutomatic,        // D-02: auto start
       ErrorControl:     mgr.ErrorNormal,
       ServiceStartName: "LocalSystem",              // D-01: LocalSystem account
       DisplayName:      cfg.Service.DisplayName,
       Description:      "自动保持 nanobot 处于最新版本", // D-03: hardcoded Chinese
     }`
8. If CreateService fails, return wrapped error.
9. Defer `svcHandle.Close()`.
10. Configure recovery policy (D-07): call `svcHandle.SetRecoveryActions()` with 3 RecoveryAction entries (first failure, second failure, subsequent failures -- all ServiceRestart with 60 second delay), and resetPeriod = 86400 (24 hours in seconds):
    ```go
    recoveryActions := []mgr.RecoveryAction{
        {Type: mgr.ServiceRestart, Delay: 60 * time.Second},
        {Type: mgr.ServiceRestart, Delay: 60 * time.Second},
        {Type: mgr.ServiceRestart, Delay: 60 * time.Second},
    }
    err = svcHandle.SetRecoveryActions(recoveryActions, 86400)
    ```
11. Also call `svcHandle.SetRecoveryActionsOnNonCrashFailures(true)` so recovery triggers on non-zero exit codes too.
12. Log success: "Windows service registered successfully" with service_name and display_name.
13. Return nil.

**Method: `UnregisterService() error`** (per D-05, D-06)
Steps in order:
1. Call `mgr.Connect()`. If error, wrap and return.
2. Defer `m.Disconnect()`.
3. Call `m.OpenService(cfg.Service.ServiceName)`. If error (service not found), log "service not registered, nothing to uninstall" and return nil (idempotent).
4. If service found: try to stop it first (D-05). Call `svcHandle.Control(svc.Stop)` to send Stop control code. Log "stopping service before uninstall".
5. After sending Stop, poll `svcHandle.Query()` in a loop (every 1 second, max 30 seconds) waiting for state `svc.Stopped`. If timeout, log warning but continue with delete.
6. Call `svcHandle.Delete()` to mark for deletion. Log "service unregistered".
7. Close the service handle.
8. Return nil. The caller (main.go) continues to console mode (D-06).

**Exported function: `RegisterService(cfg *config.Config, logger *slog.Logger) error`**
Convenience wrapper: creates `NewServiceManager(cfg, logger)`, calls `RegisterService()`, returns error.

**Exported function: `UnregisterService(cfg *config.Config, logger *slog.Logger) error`**
Convenience wrapper: creates `NewServiceManager(cfg, logger)`, calls `UnregisterService()`, returns error.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User config -> ServiceManager | Untrusted input from config.yaml (service_name, display_name) crosses into SCM API calls |
| ServiceManager -> Windows SCM | Process communicates with SCM (CreateService, DeleteService) requiring admin privileges |
| ServiceManager -> Filesystem | os.Executable() reads process binary path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48-01 | E (Elevation of Privilege) | RegisterService | mitigate | D-08: IsAdmin() checks for admin elevation before SCM operations. Non-admin gets clear error message and exits with code 1 |
| T-48-02 | S (Spoofing) | service_name config | mitigate | Phase 46 D-10: ServiceConfig.Validate() enforces alphanumeric-only service_name via regex, max 256 chars. Prevents injection into SCM API |
| T-48-03 | T (Tampering) | BinaryPathName | accept | os.Executable() returns current process path (kernel-provided). Attacker would need code execution already. Low risk for service registration |
| T-48-04 | D (Denial of Service) | UnregisterService | accept | Unregister requires admin + auto_start:false config. Legitimate admin action, not a vulnerability |
| T-48-05 | R (Repudiation) | Service operations | accept | Windows SCM logs all service create/delete to System event log. Built-in audit trail |
</threat_model>
