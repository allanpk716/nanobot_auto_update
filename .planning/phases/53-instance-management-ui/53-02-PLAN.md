<!-- API contracts for CRUD operations. All require Bearer auth. -->

POST /api/v1/instance-configs
Request body: { "name": string, "port": number, "start_command": string, "startup_timeout": number, "auto_start": boolean|null }
Response 201: { "name": string, "port": number, "start_command": string, "startup_timeout": number, "auto_start": boolean|null }
Error 422: { "error": "validation_error", "message": "Validation failed", "errors": [{ "field": string, "message": string }] }

PUT /api/v1/instance-configs/{name}
Request body: { "name": string (must match path or be empty), "port": number, "start_command": string, "startup_timeout": number, "auto_start": boolean|null }
Response 200: same as create response
Error 422: same validation error format

DELETE /api/v1/instance-configs/{name}
Response 200: { "message": "Instance {name} deleted" }
Note: Server auto-stops running instance before deletion.

POST /api/v1/instance-configs/{name}/copy
Request body: { "name": string (optional, auto-generates {source}-copy), "port": number (optional, auto-increments) }
Response 201: { full instance config of cloned instance }

Auth: Authorization: Bearer {token} from getToken() cached in authToken variable.

Existing modal system (from Plan 01): showModal(title, bodyHtml, footerHtml) shows modal; closeModal() hides it.
Existing toast system: showToast(message, type) where type = "success"|"error".
Existing refresh: loadInstances() reloads all cards from API.
</interfaces>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Form input -> API | User-provided form data (name, port, command) sent to authenticated API |
| API error -> Form | Server validation errors displayed in form (must not contain XSS payloads) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53-06 | Tampering | Form field manipulation | mitigate | Server-side validation on all CRUD endpoints (unique name/port, required fields, port range); client-side validation is UX only |
| T-53-07 | Information Disclosure | XSS via validation error messages | mitigate | Server errors displayed via textContent, not innerHTML |
| T-53-08 | Denial of Service | Delete running instance | accept | Server auto-stops instance first; confirmation dialog warns user; single-admin tool |
| T-53-09 | Tampering | CSRF on state-changing operations | accept | Single-admin localhost tool; Bearer token in header (not cookie) provides implicit CSRF protection |
</threat_model>
