Add a `SaveConfig` function to `internal/config/config.go` that persists the current in-memory Config struct to the YAML config file via viper. This function is called by the CRUD handler after modifying the in-memory config.

**Important:** The CRUD handler modifies the in-memory config obtained from `GetCurrentConfig()`, then calls `SaveConfig()` to persist. The hot-reload watcher will detect the file change and trigger async instance rebuild.

Implementation:

1. Add a `sync.Mutex` package-level var `configMu` to protect concurrent write operations (multiple API requests could try to write simultaneously).

2. Add function:
```go
// configMu protects SaveConfig from concurrent writes.
var configMu sync.Mutex

// SaveConfig persists the given Config to the YAML config file using viper.
// It acquires a mutex to prevent concurrent writes.
// The caller must ensure cfg is valid before calling SaveConfig.
//
// D-08: Uses viper.WriteConfig() to write to the same file viper loaded from.
// D-09: After writing, hot-reload watcher detects the file change (500ms debounce).
func SaveConfig(cfg *Config) error {
    configMu.Lock()
    defer configMu.Unlock()

    v := GetViper()
    if v == nil {
        return fmt.Errorf("viper not initialized")
    }

    // Set the instances key in viper to the new config value.
    // viper.WriteConfig() writes all current viper settings to the file.
    v.Set("instances", cfg.Instances)

    // WriteConfig writes to the file that was used during Load().
    // This will trigger fsnotify -> hot-reload debounce -> async instance rebuild.
    if err := v.WriteConfig(); err != nil {
        return fmt.Errorf("failed to write config file: %w", err)
    }

    return nil
}
```

3. Add `"sync"` to imports if not already present.

Notes:
- Do NOT call `GetCurrentConfig()` inside SaveConfig -- the handler passes the modified config explicitly.
- Do NOT update `globalHotReload.current` -- that is updated by the hot-reload watcher when it detects the file change.
- The mutex prevents race conditions if two API requests try to write config.yaml simultaneously.

**Request/Response JSON types** (D-05: snake_case matching YAML config keys):

```go
// instanceConfigRequest is the JSON body for create/update/copy requests.
// startup_timeout is in seconds (uint32) per D-06.
type instanceConfigRequest struct {
    Name           string `json:"name"`
    Port           uint32 `json:"port"`
    StartCommand   string `json:"start_command"`

// instanceConfigResponse is the JSON response for a single instance config.
// startup_timeout is in seconds (uint32) per D-06.
type instanceConfigResponse struct {
    Name           string `json:"name"`
    Port           uint32 `json:"port"`
    StartCommand   string `json:"start_command"`
    StartupTimeout uint32 `json:"startup_timeout"`
    AutoStart      *bool  `json:"auto_start"`

// validationErrorDetail represents a single field validation error.
type validationErrorDetail struct {
    Field   string `json:"field"`
    Message string `json:"message"`
}

// validationErrorResponse is the 422 error response (D-14).
type validationErrorResponse struct {
    Error   string                  `json:"error"`
    Message string                  `json:"message"`
```

**Handler struct:**

```go
type InstanceConfigHandler struct {

func NewInstanceConfigHandler(logger *slog.Logger) *InstanceConfigHandler {
```

**Helper methods:**

1. `toResponse(ic config.InstanceConfig) instanceConfigResponse` -- converts `InstanceConfig` (internal Duration) to response JSON (seconds uint32). `StartupTimeout` converts from `time.Duration` to seconds: `uint32(ic.StartupTimeout.Seconds())`. If `StartupTimeout == 0`, response field is `0`.

2. `toInstanceConfig(req instanceConfigRequest) config.InstanceConfig` -- converts request JSON (seconds) to internal `InstanceConfig`. `StartupTimeout` converts from seconds to `time.Duration`: `time.Duration(req.StartupTimeout) * time.Second`. If `req.StartupTimeout == 0`, leave as zero (default).

3. `writeValidationError(w http.ResponseWriter, message string, details []validationErrorDetail)` -- writes 422 response using `validationErrorResponse` struct (D-14). Calls `w.Header().Set("Content-Type", "application/json")`, then `w.WriteHeader(http.StatusUnprocessableEntity)`, then `json.NewEncoder(w).Encode(...)`.

4. `findInstanceByName(cfg *config.Config, name string) (int, *config.InstanceConfig)` -- finds instance index and pointer by name. Returns `(-1, nil)` if not found.

**Endpoint implementations:**

**HandleList (GET /api/v1/instance-configs):**
- Call `config.GetCurrentConfig()` to get live config
- If nil, return 500 via `writeJSONError`
- Convert each `InstanceConfig` to `instanceConfigResponse` via `toResponse`
- Return JSON array: `{"instances": [...]}` with status 200

**HandleGet (GET /api/v1/instance-configs/{name}):**
- Extract name: `r.PathValue("name")`
- Get config via `config.GetCurrentConfig()`
- Find instance by name via `findInstanceByName`
- If not found: 404 `{"error": "not_found", "message": "Instance 'xxx' not found"}`
- If found: return 200 with `instanceConfigResponse`

**HandleCreate (POST /api/v1/instance-configs):**
- Decode request body into `instanceConfigRequest`
- Convert to `InstanceConfig` via `toInstanceConfig`
- Validate the new instance: call `ic.Validate()`
- If validation fails: collect errors into `[]validationErrorDetail`, return 422 (D-14, D-15)
- Get current config: `config.GetCurrentConfig()`
- Check unique name: iterate `cfg.Instances`, if name matches return 422 with field error for "name"
- Check unique port: iterate `cfg.Instances`, if port matches return 422 with field error for "port"
- Append new instance to `cfg.Instances`
- Call `config.SaveConfig(cfg)` (D-07, D-08, D-09)
- If save fails: return 500
- Return 201 with the created `instanceConfigResponse`
- Log: `"Instance config created", "name", ic.Name`

**HandleUpdate (PUT /api/v1/instance-configs/{name}):**
- Extract name: `r.PathValue("name")`
- Get current config, find instance index by name
- If not found: 404
- Decode request body into `instanceConfigRequest`
- Convert to `InstanceConfig` via `toInstanceConfig`
- **Name immutability**: If `req.Name != "" && req.Name != pathName`, return 422: `{"field": "name", "message": "Instance name cannot be changed"}`. If `req.Name == ""`, use the path name.
- Validate: `ic.Validate()`
- If validation fails: 422 with field errors
- Check unique port (excluding current instance): iterate `cfg.Instances`, skip the current index, if port matches return 422
- Replace `cfg.Instances[index] = ic`
- Call `config.SaveConfig(cfg)`
- If save fails: 500
- Return 200 with updated `instanceConfigResponse`
- Log: `"Instance config updated", "name", ic.Name`

**HandleDelete (DELETE /api/v1/instance-configs/{name}):**
- Extract name: `r.PathValue("name")`
- Get current config, find instance index by name
- If not found: 404
- **Stop running instance first** (D-16): call `lifecycle.StopAllNanobots(ctx, 5*time.Second, h.logger)`. Use `context.WithTimeout(r.Context(), 30*time.Second)` for the overall operation timeout.
  - Note: `StopAllNanobots` stops ALL nanobot processes. This is the existing behavior from the hot-reload callback. For the delete case, this is acceptable since hot-reload will restart the remaining instances after the config change (D-10).
- Remove instance from slice: `cfg.Instances = append(cfg.Instances[:index], cfg.Instances[index+1:]...)`
- Call `config.SaveConfig(cfg)`
- If save fails: 500
- Return 200 with `{"message": "Instance 'xxx' deleted"}`
- Log: `"Instance config deleted", "name", name`

**HandleCopy (POST /api/v1/instance-configs/{name}/copy):**
- Extract source name: `r.PathValue("name")`
- Get current config, find source instance by name
- If not found: 404
- Clone the source instance config (copy all fields)
- **Name** (D-11): If request body is provided and contains a non-empty `name` field, use it. Otherwise, use `{sourceName}-copy`. Validate that the name does not conflict with existing instances.
- **Port** (D-12): If request body is provided and contains a non-zero `port` field, use it. Otherwise, auto-increment from `{sourcePort + 1}` until finding an unused port. Check against all existing instance ports. Max attempts: 100 (return 422 if exceeded).
- Validate the cloned instance: `ic.Validate()`
- Check unique name and port against existing instances
- Append to `cfg.Instances`
- Call `config.SaveConfig(cfg)`
- Return 201 with the new `instanceConfigResponse`
- Log: `"Instance config copied", "source", sourceName, "new_name", ic.Name, "new_port", ic.Port`

**D-13**: Copy only clones auto-updater config (InstanceConfig). Does NOT create nanobot config directory. That is Phase 52's job.

**Error handling patterns:**
- Use `writeJSONError(w, status, code, message)` for simple errors (404, 500)
- Use `writeValidationError(...)` for 422 validation errors (D-14)
- All handlers should use `json.NewDecoder(r.Body).Decode(&req)` and handle decode errors with 400 Bad Request
- Empty body on copy endpoint is valid (use defaults for name/port)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> API | Untrusted HTTP input crosses into trusted backend. All requests must be authenticated and validated. |
| API -> config.yaml | Trusted backend writes to local config file. Write operations must be serialized. |
| API -> Process Management | Delete endpoint triggers process stop. Must only stop nanobot processes, not arbitrary processes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-50-01 | Spoofing | All CRUD endpoints | mitigate | Bearer token auth via AuthMiddleware with constant-time comparison (D-03, reuses existing SEC-03) |
| T-50-02 | Tampering | POST/PUT request bodies | mitigate | Strict JSON decoding with field validation via InstanceConfig.Validate() + Config.Validate() uniqueness checks (D-15) |
| T-50-03 | Tampering | config.yaml writes | mitigate | configMu mutex serializes writes; viper.WriteConfig() handles atomic file write |
| T-50-04 | Denial of Service | DELETE endpoint | accept | StopAllNanobots stops all processes but hot-reload restarts surviving instances; acceptable for admin tool with auth |
| T-50-05 | Information Disclosure | GET responses | mitigate | Responses only contain instance config fields (name, port, command, timeout, auto_start) -- no secrets exposed |
| T-50-06 | Elevation of Privilege | POST /copy | mitigate | Copy endpoint validates name uniqueness and port range like create; no special privilege escalation path |
</threat_model>
