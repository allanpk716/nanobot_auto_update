**Important note about testing mutations:** Since `HandleCreate`, `HandleUpdate`, `HandleDelete`, and `HandleCopy` call `config.UpdateConfig()`, which uses `config.GetCurrentConfig()` internally, these mutation tests will fail because `globalHotReload` is nil in test context (UpdateConfig returns "config not initialized").

There are two approaches:
- **Approach A (recommended):** Test the handler logic by testing read-only endpoints (List, Get) with the injected config. For mutation endpoints, test them as integration tests that set up a real config file. This is simpler and more realistic.
- **Approach B:** Modify UpdateConfig to also accept an injected config reader, or add a test helper to set globalHotReload.

**Choose Approach A:** For this task, write two categories of tests:

**Category 1: Read-only handler tests (List, Get)** -- use the injected config reader. These are pure unit tests with no global state.

**Category 2: Mutation handler tests (Create, Update, Delete, Copy)** -- use integration-style tests that:
1. Create a temp config file
2. Call `config.Load()` to initialize viperInstance
3. Call `config.WatchConfig()` with no-op callbacks to initialize globalHotReload
4. Create the handler with `config.GetCurrentConfig` as the injected reader (production path)
5. Test the handler
6. Cleanup with `config.StopWatch()`

However, the WatchConfig approach starts real file watchers which can cause timing issues on Windows CI. Instead, use a simpler integration setup:

```go
func setupIntegrationTest(t *testing.T) (*InstanceConfigHandler, string) {
    t.Helper()

    tmpDir := t.TempDir()
    configPath := filepath.Join(tmpDir, "config.yaml")

    initialYAML := `api:
  port: 9999
  bearer_token: "test-token-12345678901234567890"
instances:
  - name: "test-existing"
    port: 18790
    start_command: "nanobot gateway"
    startup_timeout: 30s
    auto_start: true
`
    require.NoError(t, os.WriteFile(configPath, []byte(initialYAML), 0644))

    cfg, err := config.Load(configPath)
    require.NoError(t, err)

    logger := slog.New(slog.NewTextHandler(io.Discard, nil))

    // Start watch to initialize globalHotReload (needed for UpdateConfig -> GetCurrentConfig)
    config.WatchConfig(cfg, logger, &config.HotReloadCallbacks{})

    handler := NewInstanceConfigHandler(config.GetCurrentConfig, logger)
    token := "test-token-12345678901234567890"

    t.Cleanup(func() {
        config.StopWatch()
        // Reset viperInstance to avoid interfering with other tests
        // This requires access to the viperInstance var -- since we're in package api,
        // we can't directly set it. Instead, we rely on StopWatch() to clean up globalHotReload.
    })

    return handler, token
}
```

**Actually, the best approach to avoid flaky tests:** For mutation tests, do NOT use WatchConfig. Instead, add a test-only helper to config package. But that modifies config package which is outside this test file's scope.

**Final recommended approach:** Use the integration setup with WatchConfig BUT make tests resilient:
1. Each test that mutates config gets its own temp directory and its own config.Load() + WatchConfig()
2. Tests verify the API response immediately (the handler returns correct data regardless of hot-reload timing)
3. Tests do NOT verify file content after mutation (that is UpdateConfig's test responsibility)
4. Tests do NOT use `time.Sleep` -- they verify the API response which is synchronous
5. Cleanup calls `config.StopWatch()` to reset globalHotReload

Since each test gets its own isolated setup and teardown, there are no inter-test dependencies. The file watcher is only needed so `GetCurrentConfig()` returns non-nil (UpdateConfig requires it). The 500ms debounce timer does not affect test results because we verify the API response, not the file content.

   ```go
   func TestUpdateConfig_ConcurrentMutationsNoDataLoss(t *testing.T) {
       tmpDir := t.TempDir()
       configPath := filepath.Join(tmpDir, "config.yaml")

       initialYAML := `api:
     port: 9999
     bearer_token: "test-token-12345678901234567890"
   instances:
     - name: "existing"
       port: 18790
       start_command: "nanobot gateway"
       startup_timeout: 30s
   `
       require.NoError(t, os.WriteFile(configPath, []byte(initialYAML), 0644))

       cfg, err := Load(configPath)
       require.NoError(t, err)

       logger := slog.New(slog.NewTextHandler(io.Discard, nil))
       WatchConfig(cfg, logger, &HotReloadCallbacks{})

       var wg sync.WaitGroup
       const numGoroutines = 10
       errors := make([]error, numGoroutines)

       for i := 0; i < numGoroutines; i++ {
           wg.Add(1)
           go func(idx int) {
               defer wg.Done()
               err := UpdateConfig(func(c *Config) error {
                   c.Instances = append(c.Instances, InstanceConfig{
                       Name:           fmt.Sprintf("concurrent-%d", idx),
                       Port:           uint32(18800 + idx),
                       StartCommand:   "nanobot test",
                       StartupTimeout: 30 * time.Second,
                   })
                   return nil
               })
               errors[idx] = err
           }(i)
       }

       wg.Wait()

       // Verify no errors
       for i, err := range errors {
           assert.NoError(t, err, "goroutine %d should not error", i)
       }

       // Stop watch and reload to check persisted state
       StopWatch()
       viperInstance = nil
       newCfg, err := Load(configPath)
       require.NoError(t, err)

       // Should have 1 original + 10 new = 11 instances
       assert.Len(t, newCfg.Instances, 11, "all concurrent mutations should be preserved")
   }
   ```
