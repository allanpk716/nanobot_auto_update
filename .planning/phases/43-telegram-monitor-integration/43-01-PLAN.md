```go
// startTelegramMonitor creates and starts the Telegram monitor for this instance.
// Called after successful process start in StartAfterUpdate (D-01).
func (il *InstanceLifecycle) startTelegramMonitor() {
    monitor := telegram.NewTelegramMonitor(
        il.logBuffer,
        il.notifier,
        il.config.Name,
        telegram.DefaultTimeout,
        il.logger,
    )
    monitorCtx, cancel := context.WithCancel(context.Background())
    il.telegramMonitor = monitor
    il.monitorCancel = cancel

    go func() {
        defer func() {
            if r := recover(); r != nil {
                il.logger.Error("telegram monitor goroutine panic",
                    "panic", r)
            }
        }()
        monitor.Start(monitorCtx)
        il.logger.Debug("telegram monitor goroutine exited")
    }()

    il.logger.Info("Telegram monitor started for instance")
}
```
Key details:
- Independent `context.WithCancel(context.Background())` for monitor (not instance ctx) per RESEARCH Pitfall 1
- Panic recovery in goroutine per Claude's discretion
- Debug log on goroutine exit per RESEARCH Open Question 2
- Monitor created as LAST thing in StartAfterUpdate to avoid leak per RESEARCH Pitfall 1

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-01 | Denial of Service | monitor goroutine | mitigate | Panic recovery via defer/recover in startTelegramMonitor goroutine prevents crash propagation |
| T-43-02 | Denial of Service | monitorCancel | mitigate | Nil-check in stopTelegramMonitor prevents nil pointer dereference on double-stop |
| T-43-03 | Denial of Service | monitor goroutine leak | mitigate | Independent context.WithCancel + monitor.Stop() + monitorCancel() ensure goroutine exits on StopForUpdate |
| T-43-04 | Tampering | Notifier field | accept | Notifier is immutable after construction (D-04), set once in constructor, never replaced. Private field, no setter. |
| T-43-05 | Denial of Service | main.go creation order | accept | Notifier always non-nil (notifier.NewWithConfig returns disabled notifier, not nil). No nil pointer risk. |
