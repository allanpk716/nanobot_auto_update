**Step 1: 添加 import**

在 import 块中添加 notification 包：
```go
import (
    // ... 现有 imports ...
    "github.com/HQGroup/nanobot-auto-updater/internal/notification"
    "github.com/HQGroup/nanobot-auto-updater/internal/notifier"
)
```

**Step 2: 创建 Notifier 实例**

在网络监控启动之前，创建 Notifier 实例（使用配置文件中的 Pushover 配置）：
```go
// 创建 Notifier (MONITOR-04, MONITOR-05)
notif := notifier.NewWithConfig(
    notifier.Config{
        ApiToken: cfg.Pushover.ApiToken,
        UserKey:  cfg.Pushover.UserKey,
    },
    logger,
)
```

**Step 3: 创建并启动 NotificationManager**

在网络监控启动之后，创建并启动 NotificationManager：
```go
// 启动通知管理器 (MONITOR-04, MONITOR-05)
notificationManager := notification.NewNotificationManager(
    networkMonitor,
    notif,
    logger,
)
// 使用与网络监控相同的检查间隔
go notificationManager.Start(cfg.Monitor.Interval)
logger.Info("通知管理器已启动", "check_interval", cfg.Monitor.Interval)
```

**Step 4: 在关闭时停止 NotificationManager**

在关闭顺序中，NotificationManager 应该在网络监控停止之前停止：
```go
// 停止通知管理器（优先于网络监控）
if notificationManager != nil {
    notificationManager.Stop()
}

// 停止网络监控
if networkMonitor != nil {
    networkMonitor.Stop()
}
```

**最终启动顺序 (per D-06):**
1. API 服务器启动
2. 健康监控启动
3. 网络监控启动
4. 通知管理器启动 ← **本次添加**
5. 实例自动启动

**最终关闭顺序 (per D-06):**
1. 通知管理器停止 ← **本次添加**
2. 网络监控停止
3. 健康监控停止
4. API 服务器停止

**注意事项：**
- 确保 Notifier 在 NotificationManager 之前创建
- 确保 NotificationManager 在 NetworkMonitor 启动之后创建
- 确保 NotificationManager 在 NetworkMonitor 停止之前停止
- 使用 cfg.Monitor.Interval 作为检查间隔（与网络监控一致）
