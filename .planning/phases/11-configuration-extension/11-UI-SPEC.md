## Copywriting Contract

Configuration-related user-facing text:

| Element | Copy |
|---------|------|
| **Config validation error (Bearer Token too short)** | `api.bearer_token must be at least 32 characters for security, got {len}` |
| **Config validation error (Port invalid)** | `api.port must be between 1 and 65535, got {port}` |
| **Config validation error (Timeout too short)** | `api.timeout must be at least 5 seconds, got {timeout}` |
| **Config validation error (Monitor interval too short)** | `monitor.interval must be at least 1 minute, got {interval}` |
| **Config validation error (Monitor timeout too short)** | `monitor.timeout must be at least 1 second, got {timeout}` |
| **Config file load error** | `Configuration error: {error}\n\nPlease check your config.yaml file.\nRequired fields:\n  - api.bearer_token (at least 32 characters)\nOptional fields (have defaults):\n  - api.port (default: 8080)\n  - api.timeout (default: 30s)\n  - monitor.interval (default: 15m)\n  - monitor.timeout (default: 10s)` |
| **Startup log (success)** | `Configuration loaded successfully` (with field values, but NOT full Bearer Token) |
| **Startup log (Bearer Token safety)** | Log only `bearer_token_length: {len}`, never log the actual token value |

**YAML configuration file example:**
```yaml
# config.yaml

# HTTP API 配置
api:
  port: 8080                      # API 端口 (默认 8080)
  bearer_token: "your-secure-token-at-least-32-chars-long"  # 必需，至少 32 字符
  timeout: 30s                    # 请求超时 (默认 30s)

# 监控服务配置
monitor:
  interval: 15m                   # 检查间隔 (默认 15m)
  timeout: 10s                    # HTTP 超时 (默认 10s)
```

## Implementation Notes

Phase 11 extends the existing YAML configuration system with new fields for HTTP API and monitoring services. All validation happens at application startup, providing clear error messages to help users fix configuration issues before the service starts.

**Pre-populated from upstream artifacts:**
- Requirements (CONF-01 to CONF-06, SEC-03): Configuration validation rules and Bearer Token security
- RESEARCH.md: Validation patterns, error message format, Bearer Token security best practices
- PROJECT.md: Log format standard, error message patterns from v0.2

**Design contract is minimal** because this is a backend configuration extension with no user interface beyond the YAML file and console output.
