# 关键架构决策 (来自 RESEARCH.md)

1. 使用 `time.NewTicker()` + `select` 监听 `ctx.Done()` 和 `ticker.C`
2. `defer ticker.Stop()` 必须紧跟 `time.NewTicker()` 之后
3. 失败时记录日志但不中断服务（不返回错误）
4. 启动时立即执行一次检查（提供即时反馈）

# Ticker + Context 生命周期模式

```go
func (s *Service) Run(ctx context.Context) error {
    ticker := time.NewTicker(s.interval)
    defer ticker.Stop() // 关键: 防止 goroutine 泄漏

    // 启动时立即执行一次
    s.checkAndLog(ctx)

    for {
        select {
        case <-ctx.Done():
            s.logger.Info("监控服务停止")
            return ctx.Err()
        case <-ticker.C:
            s.checkAndLog(ctx)
        }
    }
}

func (s *Service) checkAndLog(ctx context.Context) {
    // 执行检查，记录日志，失败不返回错误
    connected, err := s.checker.CheckConnectivity(ctx)
    if err != nil {
        s.logger.Warn("连通性检查失败", "error", err.Error())
        return // 继续运行，不中断
    }
    // ...记录结果
}
```
