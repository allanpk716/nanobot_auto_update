# 关键架构决策 (来自 RESEARCH.md)

1. 使用 `context.WithTimeout()` 控制超时，不用 `http.Client.Timeout`
2. 使用自定义 `http.Client` 而非 `http.DefaultClient`
3. 2xx 状态码视为成功，非 2xx 返回 connected=false（不视为错误）

# Go 标准库模式

```go
// Context 驱动超时 (RECOMMENDED)
ctx, cancel := context.WithTimeout(parentCtx, timeout)
defer cancel()

req, err := http.NewRequestWithContext(ctx, http.MethodGet, url, nil)
resp, err := client.Do(req)

// 区分超时和其他错误
if ctx.Err() == context.DeadlineExceeded {
    // 超时处理
}
```
