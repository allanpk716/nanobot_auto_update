Define (m *TelegramMonitor) Start(ctx context.Context):
- Call m.logBuffer.Subscribe(), defer m.logBuffer.Unsubscribe(ch)
- Set m.startTime = time.Now() (TELE-08: filter entries before this)
- Enter for-select loop:
  - case entry, ok := <-ch: if !ok return; else m.processEntry(entry)
  - case <-ctx.Done(): m.mu.Lock(), stop timer if non-nil, m.mu.Unlock(), return

Define (m *TelegramMonitor) processEntry(entry logbuffer.LogEntry):
- TELE-08 filter: if entry.Timestamp.Before(m.startTime), return immediately
- m.mu.Lock(), defer m.mu.Unlock()
- switch m.state:
  - stateIdle: if IsTrigger(entry.Content) -> m.state = stateWaiting, m.startTimer()
  - stateWaiting:
    - if IsSuccess(entry.Content): m.timer.Stop(), m.state = stateIdle, go m.sendNotification("Telegram Connected", "Instance %s: Telegram bot connected successfully")
    - if IsFailure(entry.Content): m.timer.Stop(), m.state = stateIdle, go m.sendNotification("Telegram Connection Failed", "Instance %s: httpx.ConnectError detected")

Define (m *TelegramMonitor) startTimer():
- if m.timer != nil: m.timer.Stop()
- m.timer = time.AfterFunc(m.timeout, func() { ... })
  - Inside callback: m.mu.Lock(), defer m.mu.Unlock()
  - if m.state != stateWaiting: return (stale callback guard)
  - m.state = stateIdle
  - go m.sendNotification("Telegram Connection Timeout", "Instance %s: no response within %v")

Correction: sendNotification is called via `go m.sendNotification(...)`, so inside sendNotification:
```go
func (m *TelegramMonitor) sendNotification(title, message string) {
    defer func() {
        if r := recover(); r != nil {
            m.logger.Error("notification goroutine panic", "panic", r, "stack", string(debug.Stack()))
        }
    }()
    if !m.notifier.IsEnabled() {
        return
    }
    if err := m.notifier.Notify(title, message); err != nil {
        m.logger.Error("telegram notification failed", "error", err, "title", title)
    }
}
```

Define (m *TelegramMonitor) Stop():
- m.mu.Lock()
- if m.timer != nil: m.timer.Stop()
- m.mu.Unlock()
- m.cancel()

IMPORTANT design decisions (from RESEARCH):
1. State returns to stateIdle after resolution (not stateResolved) to support multiple trigger cycles — per Open Question #3 recommendation
2. Notifications are sent in goroutines to avoid blocking the subscriber read loop — per Pitfall #3
3. AfterFunc callback checks m.state != stateWaiting as stale callback guard — per Pitfall #4
4. startTimer always stops previous timer before creating new one — per Pitfall #5
5. Stop() stops timer before calling cancel() — per Pitfall #6

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-01 | Spoofing | processEntry() | accept | Log injection via crafted nanobot output: nanobot runs locally on same machine, no external injection vector. Same-trust boundary. |
| T-42-02 | Information Disclosure | sendNotification() | mitigate | Notification messages contain only instance name and connection status. No ports, file paths, commands, or credentials in notification content. Verify message templates contain no sensitive data. |
| T-42-03 | Denial of Service | Start() goroutine | mitigate | Notification send runs in separate goroutine with panic recovery. Subscriber channel capacity 100 prevents log buffer from being blocked. Timer.Stop() on all exit paths prevents timer leaks. |
| T-42-04 | Denial of Service | sendNotification() | accept | Notification spam from repeated triggers: Each trigger resets the timer (startTimer stops previous). At most 1 notification per 30s window per instance. Acceptable rate. |
</threat_model>
