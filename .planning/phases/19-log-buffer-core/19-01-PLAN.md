**实现步骤:**

1. 创建 `internal/logbuffer/buffer.go`:
   - 定义 LogEntry 结构体:
     ```go
     type LogEntry struct {
         Timestamp time.Time  // 毫秒级精度
         Source    string     // "stdout" or "stderr"
         Content   string
     }
     ```
   - 定义 LogBuffer 结构体 (不包含 subscribers 字段,Plan 02 添加):
     ```go
     type LogBuffer struct {
         mu      sync.RWMutex
         entries [5000]LogEntry  // 固定容量 5000 (BUFF-02)
         head    int             // 下一个写入位置 (0-4999)
         size    int             // 当前条目数 (0-5000)
         logger  *slog.Logger
     }
     ```
   - 实现 NewLogBuffer 构造函数:
     - 参数: `logger *slog.Logger`
     - 返回: `*LogBuffer`
     - 使用 `logger.With("component", "logbuffer")` 注入上下文 (参考 lifecycle.go:24)
     - 初始化 entries 数组 (默认零值)
   - 实现 Write 方法 (BUFF-03 线程安全, BUFF-04 FIFO 覆盖):
     - 参数: `entry LogEntry`
     - 返回: `error` (当前阶段返回 nil,预留错误处理接口)
     - 使用 `lb.mu.Lock()` 保护写入 (参考 RESEARCH.md:429)
     - 实现环形指针移动: `lb.head = (lb.head + 1) % 5000` (RESEARCH.md:434)
     - 实现 size 增长逻辑: `if lb.size < 5000 { lb.size++ }` (RESEARCH.md:435-437)
     - **不包含订阅者发送逻辑** (Plan 02 添加)
   - 实现 GetHistory 方法 (返回历史日志):
     - 参数: 无
     - 返回: `[]LogEntry` (按时间顺序)
     - 使用 `lb.mu.RLock()` 保护读取 (RESEARCH.md:457)
     - 实现 size < 5000 情况: `copy(result, lb.entries[:lb.size])` (RESEARCH.md:467)
     - 实现 size = 5000 情况: 分两段复制 `entries[head:5000]` 和 `entries[0:head]` (RESEARCH.md:470-471)

2. 创建 `internal/logbuffer/buffer_test.go`:
   - 实现 TestNewLogBuffer:
     - 验证 `NewLogBuffer(logger)` 返回非 nil
     - 验证 `lb.size == 0`
   - 实现 TestLogBuffer_Write:
     - 创建 LogEntry: `{Timestamp: time.Now(), Source: "stdout", Content: "test log"}`
     - 调用 `lb.Write(entry)`
     - 调用 `history := lb.GetHistory()`
     - 验证 `len(history) == 1`
     - 验证 `history[0].Content == "test log"`
   - 实现 TestLogBuffer_FIFO:
     - 写入 5001 条日志 (Content: "log-1", "log-2", ..., "log-5001")
     - 调用 `history := lb.GetHistory()`
     - 验证 `len(history) == 5000`
     - 验证 `history[0].Content == "log-2"` (第一条 "log-1" 被覆盖)
     - 验证 `history[4999].Content == "log-5001"`
   - 实现 TestLogBuffer_Concurrent:
     - 启动 10 个 goroutine,每个写入 100 条日志
     - 使用 `sync.WaitGroup` 等待所有 goroutine 完成
     - 验证 `lb.size == 1000`
     - **必须使用 `go test -race` 验证无数据竞态**
   - 实现 TestLogEntry_Fields:
     - 创建 LogEntry 并验证三个字段类型正确

**关键约束:**
- 容量必须固定为 5000 (BUFF-02)
- Write 和 GetHistory 必须使用互斥锁保护 (BUFF-03)
- FIFO 覆盖必须自动处理 (BUFF-04)
- LogEntry 必须包含 Timestamp、Source、Content 三个字段 (BUFF-05)
- 日志使用 `github.com/WQGroup/logger` (实际项目中使用 log/slog,参考 logging.go)

**避免:**
- 不要使用 smallnest/ringbuffer (自实现更简单)
- 不要序列化 LogEntry (直接存储对象)
- 不要在 Write 中发送到订阅者 (Plan 02 实现)
- 不要添加 subscribers 字段到 LogBuffer (Plan 02 添加)
