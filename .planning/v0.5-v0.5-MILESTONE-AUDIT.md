## E2E Flows

### Complete Flows (5/5)

1. **Application Startup Flow**
   - Steps: Config load → InstanceManager create → API server start → Health monitor start → Network monitor start → Notification manager start → Auto-start instances
   - Status: ✅ COMPLETE
   - Requirements: AUTOSTART-01, AUTOSTART-02, AUTOSTART-03, AUTOSTART-04

2. **API-Triggered Update Flow**
   - Steps: Client POST /api/v1/trigger-update → AuthMiddleware validates Bearer token → TriggerHandler.Handle() → InstanceManager.TriggerUpdate() → UpdateAll() → Stop all → UV update → Start all → Return JSON result
   - Status: ✅ COMPLETE
   - Requirements: API-01, API-02, API-03, API-04, API-05, API-06

3. **Health Monitoring Flow**
   - Steps: HealthMonitor.Start() → Periodic check (cfg.HealthCheck.Interval) → lifecycle.IsNanobotRunning() → Detect state change → Log ERROR (running→stopped) or INFO (stopped→running)
   - Status: ✅ COMPLETE
   - Requirements: HEALTH-01, HEALTH-02, HEALTH-03, HEALTH-04

4. **Network Connectivity Monitoring Flow**
   - Steps: NetworkMonitor.Start() → HTTP HEAD to google.com (cfg.Monitor.Interval, cfg.Monitor.Timeout) → Log INFO (success) or ERROR (failure) → Update ConnectivityState
   - Status: ✅ COMPLETE
   - Requirements: MONITOR-01, MONITOR-02, MONITOR-03, MONITOR-06

5. **Network State Change Notification Flow**
   - Steps: NotificationManager.Start() → Poll NetworkMonitor.GetState() → Detect state change → 1-minute cooldown timer → Send Pushover notification (recovery or failure)
   - Status: ✅ COMPLETE
   - Requirements: MONITOR-04, MONITOR-05
